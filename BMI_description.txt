1. LOYIHA HAQIDA UMUMIY MA'LUMOT
================================================================

BeLegal — O'zbekiston Respublikasi qonunchiligiga asoslangan sun'iy
intellekt yordamida ishlaydigan huquqiy ma'lumot tizimi. Platforma
foydalanuvchilarga oddiy tilda huquqiy savol berib, javobni tegishli
qonun moddalari bilan asoslangan holda olish imkonini beradi.

Muammo: Ko'pchilik fuqarolar huquqiy bilimdan mahrum, advokatga
murojaat qilish qimmat, qonun matnlari esa murakkab va tushunarsi z tilda
yozilgan. Huquqiy savodxonlik darajasi past bo'lgani uchun fuqarolar
o'z huquqlarini bilmay, ulardan foydalana olmaydi.

Yechim: BeLegal sun'iy intellekt yordamida qonun matnlarini tahlil
qilib, foydalanuvchi savoliga aniq va moddalar bilan asoslangan
javob beradi. Bu advokatni almashtirishdan ko'ra, fuqarolarga boshlang'ich
huquqiy savodxonlik berishni maqsad qiladi.

3. RAG ARXITEKTURASI (ASOSIY TEXNIK YONDASHUV)
================================================================

RAG (Retrieval-Augmented Generation) — bu sun'iy intellekt modeliga
javob berish uchun avval tegishli hujjatlarni topib, keyin ularni
kontekst sifatida modelga berish texnologiyasi.

BeLegal'da RAG quyidagicha ishlaydi:

QADAM 1 — Ma'lumotni tayyorlash (Ingest):
- O'zbekiston Konstitutsiyasi matni (moddalar.txt) yuklanadi
- Matn har bir modda bo'yicha bo'laklarga (chunks) ajratiladi
- Har bir bo'lak metadata bilan saqlanadi: modda raqami, bob nomi,
  bo'lim nomi, hujjat nomi
- Jami 155 ta modda indekslanadi

QADAM 2 — Qidiruv (Retrieval):
- Foydalanuvchi savol yozadi
- BM25 algoritmi savoldagi kalit so'zlarni barcha moddalar bilan solishtiradi
- Eng mos keladigan top-K (odatda 4) modda tanlanadi
- BM25 formula: score(D,Q) = Σ IDF(qi) * [f(qi,D)*(k1+1)] / [f(qi,D)+k1*(1-b+b*|D|/avgdl)]
  bu erda: f(qi,D) — hujjatdagi so'z chastotasi, IDF — teskari hujjat chastotasi

QADAM 3 — Generatsiya (Generation):
- Topilgan moddalar kontekst sifatida Gemini API ga yuboriladi
- Maxsus system prompt orqali model faqat berilgan kontekst asosida
  javob berishi majburlashtiriladi
- Model javobni strukturalangan formatda beradi:
  * Asosiy javob
  * Huquqiy asoslar (har bir modda uchun)
  * Xulosa
- Javob token-by-token streaming orqali foydalanuvchiga uzatiladi (SSE)

QADAM 4 — Manbalarni ko'rsatish:
- Javob ostida foydalanilgan moddalar card ko'rinishida chiqadi
- Har bir manbaga bosilsa, to'liq hujjat sahifasiga o'tib,
  tegishli modda sariq rangda ajratib ko'rsatiladi

6. API ENDPOINTLARI
================================================================

AUTENTIFIKATSIYA:
POST /api/auth/register  — Yangi foydalanuvchi ro'yxatdan o'tish
  Kirish: { email, full_name, password }
  Chiqish: { access_token, token_type }

POST /api/auth/login     — Tizimga kirish
  Kirish: { email, password }
  Chiqish: { access_token, token_type }

GET  /api/auth/me        — Joriy foydalanuvchi ma'lumotlari
  Header: Authorization: Bearer <token>
  Chiqish: { id, email, full_name, plan, created_at }

CHAT:
POST /api/chat           — Savol berish (SSE streaming)
  Kirish: { query: string, k: number }
  Chiqish: Server-Sent Events oqimi:
    data: {"type": "sources", "sources": [...]}
    data: {"type": "token", "content": "..."}
    data: [DONE]

HUJJATLAR:
GET  /api/documents      — Barcha hujjatlar strukturasi
  Chiqish: { documents: [{ name, bobs: [{ name, bolim, moddalar: [...] }] }] }

GET  /api/health         — Server holati tekshirish

7. XAVFSIZLIK MEXANIZMLARI
================================================================

1. Parol shifrlash: bcrypt algoritmi (salt rounds=12)
   — Har bir parol uchun tasodifiy salt generatsiya qilinadi
   — Rainbow table hujumlaridan himoya

2. JWT autentifikatsiya:
   — Token muddati: 30 kun
   — HS256 algoritmi bilan imzolangan
   — Har so'rovda Authorization header tekshiriladi

3. CORS himoyasi:
   — Faqat localhost:5173 va localhost:3000 ga ruxsat

4. Hallucination oldini olish:
   — System prompt orqali model faqat berilgan kontekst asosida
     javob berishga majburlashtiriladi
   — Agar javob topilmasa, aniq xabar beriladi

5. Protected Routes:
   — Frontend tomonida autentifikatsiya tekshiruvi
   — Token bo'lmasa, landing sahifasiga yo'naltiriladi
